cabinFever · marbling prototypes

Six frozen prototypes plus two short-lived spikes for the SDF shape direction. All static HTML; nothing under src/ is involved.

Marker variants (P2 children — closest path)

  • Markers · cross-pool color carry (M1) spike M1
    A marker pushed into another pool’s territory contributes its color to the rendered output, not just its home pool’s field. Green marker dragged into red region renders as visible green tint on red base. CROSS_POOL_STRENGTH = 1.5.
  • Markers · anisotropic streaks (M2) spike M2
    Each marker carries orientation + aspect. Field contribution is an elongated Gaussian. Rake/comb sets orientation along stroke, increases aspect; aspect decays slowly back to circular. Combed pigment reads as flowing streaks, not displaced dots.
  • Markers · per-pool trail painting (M3) spike M3
    Each pool owns a 128×128 scar canvas. As markers move, they paint additive Gaussian splats along their path. Scar decays at ~2.3s half-life. Field eval samples scar for extra density contribution — streaks persist for ~5s after stroke ends. Agent verified via playwright.

Metaball rake variants (preserve look, fix tool-carve)

  • Metaball · lobe fission on rake (P1) spike P1
    On first rake/comb contact a pool replaces its sparse lobes with K=12 sub-lobes per parent (~10 px radius). Existing per-lobe scar-follow then works because lobe pitch drops below tine pitch. Stays fissioned for the rest of the session.
  • Metaball · inner markers (P2) spike P2
    Each pool seeds 120 internal marker particles that contribute small extra field bumps (sigma 7 px, weight 0.28). Rake/comb pushes markers, leaves lobes alone — pool boundary stays coherent, ripples come from marker streaks.
  • Metaball · per-pool scar field (P3) spike P3
    Each pool owns a 128×128 vector displacement texture in pool-local UV. Rake paints into it; field eval samples the scar for sample-point warp. Pool-scoped so warp can’t wander outside the lobe support (the failure mode of the original global field-warp attempt).

Architecture spikes (non-pareil bake-off)

  • Advection · WebGL2 stable-fluids (path G) spike G
    Stam-style semi-Lagrangian advection on ping-pong RGBA16F textures (512×512 sim grid). Velocity + dye fields, no pressure projection. Comb = N tine velocity splats in one frame; non-pareil emerges from the advection step rather than being authored. Recommended path per the comparative analysis.
  • Math operators · Lu et al. 2012 (path H) spike H
    Pigment as a flat array of ~20k color points. Every tool is a deterministic closed-form transform: drop = area-preserving radial push, comb = summed-Gaussian shear across N tines. Direct ImageData stamping; no field, no simulation.
  • Tracer particles · dense impulse tracers (path D) spike D
    Scale-up of the particle-displacement baseline. Up to 120k typed-array tracers, spatial hash for broad-phase, same summed-Gaussian comb operator as path H. Honest underdog — the open question is whether the JS+Canvas2D budget supports enough particle density for ripples to read as continuous.

Synthesis

  • Fluid particles · SDF + particle displacement + metaballs synthesis
    Particle-displacement physics (impulses, separation, wet curl) drives a flat list of independent particles. Each drop’s particles sum into a metaball field with anisotropic + tendril domain warp; cells are owned by the strongest-contributing drop, with two-pigment muddy blend at seams and inner sub-metaballs for interior swirls. SDF-style surface treatment painted per cell: fbm paper grain, vein streaks, granulation specks, rim darken, and stain bloom into nearby paper. Mid-press preview impulse so existing pigment visibly reacts while you hold.

Earlier spikes (under evaluation)

  • Shader / SDF — Spike A spike A
    In-place fixes to the SDF prototype: coordinate-space domain warp, two-owner pigment blend at seams, growth-aware warp amplitude.
  • Hybrid · raster ownership + shader surface — Spike B spike B
    Raster sim drives owner ids into a texture; shader resolves seams from cardinal-neighbour disagreement and applies SDF-style surface treatment with a domain-warped lookup.
  • Metaball pools — refined refine
    Perf: direct pixel writes (replaces per-cell Canvas2D fill). Visual: two-pigment muddy seam blend; coordinate-space domain warp. Marbling-feel: volume-conserving displacement (new drops push existing pools’ lobes outward, asymmetrically — pools stretch), inner sub-metaballs for fluid interior color swirls (“metaball within metaball”), anisotropic boundary with tendril noise, satellite spawning at contact zones (older pigment squeezed sideways when a new drop lands on it), and mid-press preview displacement so neighbours deform live as you hold. Compare back-to-back against the frozen baseline below.
  • Metaball pools — hi-res refine
    Same simulation as the refined variant, but the cell write does bilinear interpolation of field strength across each cellSize×cellSize block instead of stamping a solid color. Rim alpha, rim-darken, wash, and seam blend recompute per pixel from the interpolated strength — rims read smoothly antialiased against paper, interior gradients no longer step at cell boundaries. Field grid is also finer (tiers {2,3,4} instead of {4,6,8}). Sharp ownership seam between two pools preserved by clamping bilerp on different-owner neighbors. Same tools, same physics; compare rim sharpness side-by-side against the refined variant above.
